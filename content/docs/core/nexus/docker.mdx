本文档将指导你通过 Docker 快速部署 **FGate Nexus**。

<Callout type="info">
  **前置要求：** 请确保你的服务器已安装 [Docker](https://docs.docker.com/get-docker/) 和 [Docker
  Compose](https://docs.docker.com/compose/install/)（可选但推荐）。
</Callout>

## 镜像信息

| 字段       | 说明                                 |
| ---------- | ------------------------------------ |
| 镜像地址   | `wlingxd/fgatenexus`                 |
| 标签格式   | `wlingxd/fgatenexus:<tagname>`       |
| 暴露端口   | `3000`（Web UI）、`5140`（服务端口） |
| 持久化目录 | `/app/config`、`/app/data`           |

## 快速开始

<Steps>

<Step>

### 拉取镜像

将 `<tagname>` 替换为你需要的版本号，例如 `latest` 或 `0.3.4`：

```bash
docker pull wlingxd/fgatenexus:<tagname>
```

拉取最新版本：

```bash
docker pull wlingxd/fgatenexus:latest
```

</Step>

<Step>

### 创建本地目录

在宿主机上创建用于持久化数据的目录并进入：

```bash
mkdir -p ./fgatenexus/config ./fgatenexus/data
cd ./fgatenexus
```

</Step>

<Step>

### 启动容器

<Tabs items={['Docker CLI', 'Docker Compose']}>

<Tab value="Docker CLI">

```bash
docker run -d \
  --name fgatenexus \
  --restart unless-stopped \
  -p 3000:3000 \
  -p 5140:5140 \
  -v ./config:/app/config \
  -v ./data:/app/data \
  wlingxd/fgatenexus:<tagname>
```

</Tab>

<Tab value="Docker Compose">

在 `fgatenexus` 目录下创建 `docker-compose.yml` 文件：

```yaml
services:
  fgatenexus:
    image: wlingxd/fgatenexus:<tagname>
    container_name: fgatenexus
    restart: unless-stopped
    ports:
      - "3000:3000"
      - "5140:5140"
    volumes:
      - ./config:/app/config
      - ./data:/app/data
```

然后执行：

```bash
docker compose up -d
```

</Tab>

</Tabs>

</Step>

<Step>

### 验证运行状态

```bash
docker ps | grep fgatenexus
```

查看实时日志：

```bash
docker logs -f fgatenexus
```

看到类似如下输出，说明服务已成功启动：

```log
[MIGRATION] 检查并执行数据库迁移...
[SUCCESS] 数据库准备就绪
INFO: 配置文件路径：...\config\appsettings.json
Listening on http://127.0.0.1:3000/...
INFO: Koishi 服务启动 http://127.0.0.1:5140
```

</Step>

<Step>

### 访问服务

打开浏览器，访问：

```
http://<你的服务器IP>:3000
```

</Step>

</Steps>

## 端口说明

| 端口   | 用途            | 是否必须开放 |
| ------ | --------------- | ------------ |
| `3000` | Web 管理界面    | ✅ 推荐      |
| `5140` | Koishi 服务端口 | ✅ 必须      |

<Callout type="warn">请确保服务器防火墙已放行以上端口，否则外部无法访问服务。</Callout>

## 数据持久化

容器声明了两个挂载卷，**务必将其映射到宿主机目录**，否则容器重建后数据将丢失：

| 容器路径      | 说明                                   |
| ------------- | -------------------------------------- |
| `/app/config` | 存储配置文件，如 `appsettings.json` 等 |
| `/app/data`   | 存储运行时数据、数据库等               |

## 更新镜像

<Steps>

<Step>

### 拉取新版本镜像

```bash
docker pull wlingxd/fgatenexus:<new-tagname>
```

</Step>

<Step>

### 停止并删除旧容器

```bash
docker stop fgatenexus && docker rm fgatenexus
```

</Step>

<Step>

### 启动新容器

重新执行[启动容器](#启动容器)步骤，使用新的 `tagname` 即可。由于数据已挂载到宿主机，更新不会丢失任何数据。

</Step>

</Steps>

## 常用命令速查

```bash
# 启动容器
docker start fgatenexus

# 停止容器
docker stop fgatenexus

# 重启容器
docker restart fgatenexus

# 查看日志（最近 100 行）
docker logs --tail 100 fgatenexus

# 进入容器 Shell
docker exec -it fgatenexus sh

# 删除容器（数据不受影响）
docker rm -f fgatenexus
```
